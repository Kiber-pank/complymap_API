# Complymap_API

Сервер для работы с данными деклараций и сертификатов complymap_API. Предоставляет REST API с фильтрацией, пагинацией и подтягиванием справочников через JOIN.

## Стек технологий

- Node.js 20+
- TypeScript
- Fastify
- PostgreSQL 17+
- Zod (валидация запросов)
- Pg (пул соединений)

## Требования

- Node.js версии 20 или выше
- PostgreSQL 17 или выше
- Настроенные переменные окружения (см. раздел ниже)

## Установка и запуск

1. Склонируйте репозиторий:
```bash
git clone https://github.com/Kiber-pank/complymap_API.git
cd fsa-api
```
2. Установите зависимости:
```bash
npm install
```

3. Cоздайте файл окружения:
```bash
cp .env.example .env
```
Отредактируйте .env, указав реальные доступы к базе данных.

4. Запустите сервер в режиме разработки:
```npm run dev```
Сервер будет доступен по адресу http://localhost:3000.

## Переменные окружения

| Переменная    | Описание                 | Пример                        |
| ------------- | ------------------------ | -----------------------------:|
| PG_HOST       | Адрес сервера PostgreSQL |  localhost или db.example.com |
| PG_PORT       | Порт PostgreSQL          |                          5432 |
| PG_USER       | Пользователь базы данных |                  api_readonly |
| PG_PASSWORD   | Пароль пользователя      |               secure_password |
| PG_DB         | Имя базы данных          |                        fsa_db |
| PORT          | Порт HTTP-сервера        |                          3000 |
| NODE_ENV      | Окружение                |    development или production |

## Структура проекта

src/
├── config/           # Валидация переменных окружения
├── db/               # Настройка пула соединений PostgreSQL
├── modules/
│   └── declarations/ # Маршруты, сервисы и схемы для деклараций
├── plugins/          # Глобальные плагины (обработчик ошибок, rate-limit)
├── routes/           # Системные маршруты (healthcheck)
├── app.ts            # Сборка приложения Fastify
└── server.ts         # Точка входа и запуск HTTP-сервера

## API Эндпоинты

| Метод | Путь              | Описание                                             |
| ------| ------------------| ----------------------------------------------------:|
| GET   | /health           | Проверка работоспособности сервера и соединения с БД |
| GET   | /api/declarations | Получение списка деклараций с фильтрами и пагинацией |

Полная документация OpenAPI доступна после запуска по адресу: http://localhost:3000/docs

## Разработка

- Код написан на TypeScript с включённым strict режимом
- Валидация входящих запросов выполняется через Zod
- Ошибки перехватываются глобальным обработчиком и логируются в структурированном формате
- Запросы к базе данных параметризованы, SQL-инъекции исключены архитектурой
